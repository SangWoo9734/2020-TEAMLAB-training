### 1. CSS를 HTML에 적용하는 세가지 방법의 장단점은 무엇인가요?
#### 1.1 Inline Style Sheet
Inline Style Sheet는 HTML 태그에 직접 style 속성에 CSS 코드를 직접 넣어 적용하는 방법이다.
> ``` <p style="background-color : gray>Hello world.</p> ```
CSS가 HTML 태그의 속성과 값ㅍ으로 들어간다. 꾸미는 것에 한계가 있고, 재사용성이 떨어진다.
#### 1.2 Internal Style Sheet
Internal Style Sheet는 HTML 문서 내에 style 태그로 스타일 코드를 넣는 방법이다.
> ```<style> hl { color : blue; background-color : gray;} </style>```
동일한 태그에 대해 한번에 적용할 수 있다는 장점이 있지만, 다른 HTML 문서에 동일하게 적용할 수 없다.
#### 1.3 Linking Style Sheet
Linking Style Sheet는 HTML 파일과 별도로 CSS파일을 만들어 HTML 문서와 연결하여 적용하는 방법이다.
> ``` <link rel="stylesheet" href="style.css"> ```
여러 태그에 대해서 한번에 적용가능하고, link로 연결하기만 하면 여러 파일에도 한번에 적용가능하다.

### 2. 여러 개의 CSS 규칙이 한 개의 대상에 적용될 때. 어떤 규칙이 우선순위를 가지게 되나요?
CSS 규칙이 여러 대상에게 적용 되었을 때의 우선순위는 다음과 같다.
1. 속성값 뒤에 `!important`가 붙어있는 것
1. Inline Style Sheet로 적용된 경우
1. 선택자가 `#ID`로 적용된 경우
1. 선택자가 `.Class`로 적용된 경우
1. 선택자가 태그 이름으로 적용된 경우
1. 상위 객체의 상속된 속성

### 3. 어떤 박스가 `position: absolute;`인 속성을 갖는다면, 그 위치의 기준점은 어디가 되나요? 
`position:absolute;`의 속성을 갖게된다면, 부모 엘리먼트에 속해 있어도 독립적인 배치를 가진다.
부모 엘리먼트와 겹쳐서 배치도 가능하다.<br>
부모 엘리먼트가 `position:relative;`의 속성을 갖고 있다면 부모 엘리먼트를 기준으로 자유롭게 배치 가능하며,
만약 부모 엘리먼트가 `position` 속성을 가지고 있지않다면, 화면 전체에 자유롭게 배치 가능하다.

### 4. 가로나 세로로 여러 개의 박스가 공간을 채우되, 그 중 한 개의 박스만 가변적인 크기를 가지고 나머지 박스는 고정된 크기를 갖게 하려면 어떻게 해야 할까요?
각 박스에 `flex-basis` 속성을 추가하고 일정 공간을 부여한다. 그 후 변화하지 않게 하기위해 `flex-shirink:0;` 속성을 부여하면 고정된 크기를 가지게된다.
가변적인 크기를 가져야하는 박스의 속성에는 `flex-shrink:1` 속성과 'flex-grow:1'를 부여햐여 화면 페이지가 늘어나고 줄어듬에 따라 변화할 수 있게 된다.
### 5. `float` 속성은 왜 좋지 않을까요?
`float` 속성은 위치 관련 속성으로 텍스트의 흐름 속에 개체를 넣는 방식이다. 이 `float`로 선언된 속성은 페이지의 흐름 내에서 제외되며 다른 요소에 영향을 주지 못한다.
<br>
`float` 속성은 다음과 같은 문제점을 가지고 있다.
1. 밀려내려감
`float' 속성을 가진 요소는 요소의 크기가 고정되어있기 때문에 그 요소보다 작은 공간에 들어가게 되면 그 아래 공간으로 밀려내려가게 된다. 이로 인해 다른 요소들의
위치까지 조절해야하는 번거로움이 발생할 때가 있다.
1. 요소들의 전체 너비를 계산에 모호하다.
`float`속성들로 화면 레이아웃을 구성하게 되면 화면의 너비를 고려할 때 float의 Margin 값과 Padding값을 고려하여야 한다.
### 6. Flexbox(Flexible box)를 사용할 때의 한계점은 무엇인가요?
1. 현재 최신 브라우저에는 Flexbox를 지원하지만 일부 오래된 브라우저(예를들어 IE의 이전 버전)에는 지원하지 않는다. 이 점은 현재의 단점이긴 하지만 시간이 지나면서 해결될 사항으로 보인다.
1. CSS 코드 자체가 직관적이지 않다. CSS 코드만을 보고 어떤 요소들이 어떻게 어디에 배치되는지 실제 레이아웃을 보지않고는 파악하기 힘들다는 점이 있다.
1. 한 개의 열 또는 행만 배치 가능하게 되어있다. 여러 행을 배치하려면 여러개의 Flexible Boxㄹㄹ 사용해야한다는 단점이 있다.
